# Cellbig CMS
- 셀빅이 개발하여 납품하는 다양한 제품의 콘텐츠를 관리하는 통합 CMS 시스템

## Cellbig Content 개요
- 셀빅은 다양한 실감형 콘텐츠 제품들을 각종 전시장, 박물관, 유치원, 학교등에 납품
- 각 제품들은 고유의 콘텐츠 어플리케이션을 포함
- 제품(Product)는 1개 이상의 어플리케이션으로 구성된다.
  - 제품은 내부에 1개 이상의 하위 콘텐츠를 포함하고 있다.
  - 각 콘텐츠는 고유의 Content ID를 가진다.
- 패키지(Package)는 1개의 제품 내부의 하위 콘텐츠의 그룹이다.
  - 예를 들어, 샌드 크래프트는 총 24종의 하위 콘텐츠를 포함하여, 11종만 제공하는 패키지와 24종을 모두 제공하는 패키지로 구분될 수 있다.
  - CMS 시스템에서는 패키지 단위로 콘텐츠를 관리한다.
  - 각 패키지는 고유의 Package ID를 가진다.
- 디바이스(Device)
  - CMS 시스템에서 관리하는 제품이 설치된 물리적 장치
  - 디바이스는 중심이 되는 구동 PC와 연결된 주변 장치(터치 모니터, 프로젝터, 센서 등)로 구성된다.
  - 각 디바이스는 고유의 Device ID를 가진다.
  - 제품에 따라 2개 이상의 디바이스로 구성된 경우도 있다.
  - 각 디바이스는 CMS 서버에 등록되어야 하며, CMS 서버와 통신하여 콘텐츠 인증 및 다운로드를 수행한다.
  - 납품된 제품의 패키지에 따라 디바이스에 제공되는 콘텐츠가 달라질 수 있다.
  - 콘텐츠(Content)
    - 제품 내부에 포함된 개별 콘텐츠 어플리케이션
    - 각 콘텐츠는 고유의 Content ID를 가진다.
    - 제품에 따라 콘텐츠 콘테이너 역할을 수행하는 콘텐츠와 하위의 세부 콘텐츠로 구성된 경우도 있다.
    - 콘텐츠는 아래의 타입을 갖는다.
      - 콘텐츠 콘테이너
        - 단일 어플리케이션
        - 하위 콘텐츠를 관리하는 콘테이너 콘텐츠
        - 예: 키즐 글로벌의 경우, 키즐 글로벌 콘테이너가 별도도 콘텐츠로 하위 콘텐츠를 관리하며, 하위에 30여종의 세부 콘텐츠를 포함
      - 하위 콘텐츠
        - 단일 어플리케이션으로 구성된 콘텐츠
        - 콘텐츠 콘테이너에 의해 관리된다.
      - 혼합 콘텐츠
        - 단일 어플리케이션으로 구성된 콘텐츠
        - 콘텐츠 콘테이너가 별도로 없으며, 콘테이너와 하위 콘텐츠가 단일 어플리케이션을 구성되어 있다.
        - 예: 샌드 크래프트, 판타 슈터 등
      - 단독 콘텐츠
        - 단일 어플리케이션으로 구성된 콘텐츠
        - 콘테이너가 없고, 단독으로 운영되는 콘텐츠
        - 예: 판타 드로우잇, 판타 스케치 등
- 스토어(Store)
  - 디바이스가 설치된 장소에 따라 여러 스토어로 구분
  - 예: 특정 박물관, 유치원, 학교 등
  - 스토어는 여러 디바이스를 포함할 수 있다.
  - 스토어는 고유의 Store ID를 가진다.
  - 디바이스는 특정 스토어에 할당되어 관리된다.
- 지역(LocalArea)
  - 스토어를 묶어서 관리하는 스토어의 그룹 단위
  - 예: 특정 도시, 국가, 대륙 등
  - 지역은 여러 스토어를 포함할 수 있다.
  - 지역은 고유의 LocalArea ID를 가진다.
  - 스토어는 특정 지역에 할당되어 관리된다.
  - 지역에는 지역관리자가 할당될 수 있으며, 지역관리자는 해당 지역의 스토어 및 디바이스를 관리할 수 있다.

## 주요 기능
### 콘텐츠 관리
- 다양한 유형의 콘텐츠 어플리케이션을 통합 관리
- CMS 서버
  - 서버 관리 페이지를 통한 콘텐츠의 생성, 수정, 삭제 기능 제공
  - 콘텐츠 버전 관리 및 이력 추적 기능 지원
  - 디바이스, 제품, 패키지 관리 기능 제공
  - 메타데이터 관리 기능 제공
- 콘텐츠 클라이언트
  - CMS 서버와 연동하여 콘텐츠 다운로드 및 업데이트 기능 제공
### 보안 인증
- 상시 인증 및 One time 인증 기능 제공
- 상시 인증
  - 제품의 설치 환경이 상시적으로 인터넷 연결이 가능한 경우
  - 콘텐츠 시작시 CMS 서버와 통신하여 인증
    - 콘텐츠의 고유 Content ID + PC의 MAC Address 기반 인증
    - 인증 횟수를 서버에서 기록하여 콘텐츠의 실행 횟수에 따른 콘텐츠 선호도 분석 가능
- One time 인증
    - 제품의 설치 환경이 인터넷 연결이 불안정하거나 불가능한 경우
    - 콘텐츠 최초 실행시 CMS 서버와 통신하여 1회성 인증
