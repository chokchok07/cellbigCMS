# CMS 웹페이지 UI 플로우

## 목적
관리자·운영자가 CMS를 통해 콘텐츠, 패키지, 제품, 디바이스를 일관되고 안전하게 관리하고 배포할 수 있도록 화면별 UI 흐름과 주요 상호작용을 정의한다.

## 대상 사용자
- 시스템 관리자 (Admin)
- 콘텐츠 관리자 (Editor)
- 배포/운영 담당자 (Operator)

## 핵심 원칙
- 단일 진입점: 로그인 → 대시보드 → 작업별 화면
- 상태 기반 워크플로우: Draft → Staged(Test) → Published → Rollback
- 권한분리: 역할에 따라 메뉴·액션 가시성 제어
- 명확한 피드백: 비동기 작업에 작업 ID와 진행 상태 제공

## 화면 구조 요약
- 상단 네비게이션: 제품(프로젝트) 선택, 전역 검색, 알림, 사용자 메뉴
- 사이드바: 대시보드, 콘텐츠, 패키지, 제품, 디바이스, 리포트, 설정
- 메인 영역: 리스트뷰 / 디테일뷰 / 워크플로 위젯

## 주요 화면 및 플로우

### 1) 로그인 (Auth)
- 입력: 이메일/아이디, 비밀번호, 2FA(옵션)
- 성공: 권한 기반 대시보드로 리다이렉트
- 실패: 에러 토스트 + 잠금/재시도 안내

### 2) 대시보드
- 구성: 배포 상태 요약, 최근 활동, 알림, 빠른 작업(콘텐츠 업로드 등)
- 액션: 항목 클릭 → 상세 화면(콘텐츠/패키지/디바이스)으로 이동

### 3) 콘텐츠 라이브러리 (Content Library)

#### 콘텐츠 리스트 ([wireframe/content-list.md](./wireframe/content-list.md))
콘텐츠 관리의 중심 화면으로, 등록된 모든 콘텐츠를 확인하고 관리합니다.

**주요 기능:**
- **검색 및 필터링**: 키워드 검색, Type/Status/Product/Category별 필터링
- **상태 표시**: Draft ⚪ / Staged 🟡 / Published 🟢 / Deprecated ⚫
- **일괄 작업 (Bulk Actions)**: 여러 콘텐츠 선택 후 Publish / Archive / Delete 실행
- **개별 액션**: Edit (편집) / Del (삭제) / Ver (버전 관리)
- **페이지네이션**: 대량 콘텐츠 효율적 탐색
- **정렬**: 제목, 버전, 상태 등 컬럼별 정렬

**주요 플로우:**
1. 콘텐츠 라이브러리 진입 → 전체 콘텐츠 리스트 표시
2. 필터/검색으로 원하는 콘텐츠 찾기
3. 개별 콘텐츠 클릭 또는 액션 버튼으로 상세 작업 수행:
   - [Edit]: 콘텐츠 에디터로 이동
   - [Ver]: 버전 등록/히스토리 화면으로 이동
   - [Del]: 삭제 (확인 모달)
4. 여러 콘텐츠 선택 후 일괄 작업 수행
5. [+ New Content] 버튼으로 신규 콘텐츠 생성

#### 콘텐츠 생성·편집 플로우 ([wireframe/content-editor.md](./wireframe/content-editor.md))
1. 콘텐츠 리스트에서 [+ New Content] 클릭 → 에디터 오픈(모달 또는 전용 페이지)
2. 메타 입력: 제목(JSON 다국어), 설명, Type, Category, Product ID, entryPoint, Thumbnail
3. 저장: contents 테이블에 메타데이터 저장 (임시 Draft 상태)
4. 버전 등록: [신규 버전 등록] 버튼 → 버전 등록 화면으로 이동
5. ZIP 아티팩트 업로드 → checksum 생성 및 검증 → content_versions 테이블에 저장
6. Staging 선택 시: 특정 테스트 디바이스/테스트 그룹 지정 → 스테이징 배포
7. 테스트 확인 후 Publish(또는 패키지 포함 후 배포)

### 4) 버전 관리
- 각 콘텐츠는 `content_version`으로 관리
- 버전 생성 시 변경 로그(Release note) 입력 권장
- 버전별 Rollback 지원(별도 확인 모달 및 영향 분석)

### 5) 제품 관리 (Product)

#### 제품 리스트 ([wireframe/product-list.md](./wireframe/product-list.md))
제품 관리의 중심 화면으로, 등록된 모든 제품을 확인하고 관리합니다.

**주요 기능:**
- **검색 및 필터링**: 키워드 검색
- **통계 표시**: 연결된 패키지 수, 설치된 디바이스 수
- **일괄 작업 (Bulk Actions)**: 여러 제품 선택 후 Delete 실행
- **개별 액션**: Edit (편집) / View (상세보기) / Del (삭제)

**주요 플로우:**
1. 제품 라이브러리 진입 → 전체 제품 리스트 표시
2. 필터/검색으로 원하는 제품 찾기
3. 개별 제품 클릭 또는 액션 버튼으로 작업 수행:
   - [Edit]: 제품 편집 화면(product-editor)으로 이동
   - [View]: 제품 상세 페이지로 이동
   - [Del]: 삭제 (확인 모달, 의존성 검사)
4. [+ New Product] 버튼으로 제품 등록 화면(product-editor)으로 이동

#### 제품 상세 ([wireframe/product-detail.md](./wireframe/product-detail.md))
개별 제품의 상세 정보, 연결된 패키지, 포함된 콘텐츠를 확인하고 관리하는 화면입니다.

**주요 섹션:**
- **기본 정보**: Product ID, 제품명, 설명, 기본 패키지, 생성/수정일
- **연결된 패키지**: 이 제품에 연결된 패키지 목록 (추가/제거 가능)
- **포함된 콘텐츠**: 제품의 패키지에 포함된 모든 콘텐츠 목록

**주요 플로우:**
1. 제품 정보 편집 → 제품명, 설명, 기본 패키지 수정
2. 패키지 추가 → 기존 패키지 선택 및 연결
3. 패키지 제거 → 연결 해제 (기본 패키지 제거 시 경고)
4. 제품 삭제 → 의존성 검사 후 삭제 (연결된 패키지 있으면 차단)

#### 제품 등록/편집 ([wireframe/product-editor.md](./wireframe/product-editor.md))
신규 제품을 등록하거나 기존 제품 정보를 편집하는 화면입니다.

**주요 기능:**
- **기본 정보 입력**: Product ID (자동생성/수동), Product Name (필수), Description, Default Package
- **Product ID 자동 생성**: "product-{랜덤문자열}" 형식으로 자동 생성
- **패키지 연결 관리**: 편집 모드에서 연결된 패키지 추가/제거 가능
- **검증**: 필수 항목 검사, Product ID 중복 검사, 형식 검증

**주요 플로우:**
1. 신규 등록: Product List에서 [+ New Product] → 기본 정보 입력 → 저장 → Product Detail로 이동
2. 편집: Product List/Detail에서 [Edit] → 정보 수정 및 패키지 관리 → 저장 → Product Detail로 복귀
3. Product ID는 신규 시 자동생성 또는 수동 입력, 편집 시 읽기전용
4. 변경사항 있을 때 취소 또는 페이지 이탈 시 확인 모달 표시

### 6) 패키지 관리 (Package)

#### 패키지 리스트 ([wireframe/package-list.md](./wireframe/package-list.md))
패키지 관리의 중심 화면으로, 등록된 모든 패키지를 확인하고 관리합니다.

**주요 기능:**
- **검색 및 필터링**: 키워드 검색, Product/Status(Published/Draft)별 필터링
- **상태 표시**: Published 🟢 / Draft 🟡
- **통계 표시**: 포함된 콘텐츠 수
- **일괄 작업 (Bulk Actions)**: 여러 패키지 선택 후 Delete / Publish / Unpublish 실행
- **개별 액션**: View (상세보기) / Edit (편집) / Delete (삭제)

**주요 플로우:**
1. 패키지 라이브러리 진입 → 전체 패키지 리스트 표시
2. 필터/검색으로 원하는 패키지 찾기
3. 개별 패키지 클릭 또는 액션 버튼으로 작업 수행:
   - [View]: 패키지 상세 정보 확인
   - [Edit]: 패키지 편집 화면(package-editor)으로 이동
   - [Delete]: 삭제 (확인 모달)
4. [+ New Package] 버튼으로 패키지 등록 화면(package-editor)으로 이동

#### 패키지 등록/편집 ([wireframe/package-editor.md](./wireframe/package-editor.md))
신규 패키지를 등록하거나 기존 패키지 정보를 편집하는 화면입니다.

**주요 기능:**
- **기본 정보 입력**: Package ID (자동생성/수동), Product (필수), Name (필수), Description, Status (Draft/Published)
- **Package ID 자동 생성**: "pkg-{랜덤문자열}" 형식으로 자동 생성
- **콘텐츠 관리**: 편집 모드에서 연결된 콘텐츠 추가/제거 가능
- **검증**: 필수 항목 검사, Package ID 중복 검사, 형식 검증

**주요 플로우:**
1. 신규 등록: Package List에서 [+ New Package] → 기본 정보 입력 → 저장 → Package List로 복귀
2. 편집: Package List에서 [Edit] → 정보 수정 및 콘텐츠 관리 → 저장 → Package List로 복귀
3. Package ID는 신규 시 자동생성 또는 수동 입력, 편집 시 읽기전용
4. Status 변경 시 경고 (Draft → Published: 클라이언트 조회 가능)
5. 변경사항 있을 때 취소 또는 페이지 이탈 시 확인 모달 표시

### 7) 디바이스 관리 (Device)

#### 디바이스 리스트 ([wireframe/device-list.md](./wireframe/device-list.md))
디바이스 관리의 중심 화면으로, 등록된 모든 디바이스를 확인하고 관리합니다.

**주요 기능:**
- **검색 및 필터링**: 키워드 검색(이름, ID, MAC), Product/Store별 필터링
- **정보 표시**: 디바이스 ID, 이름, 제품, 매장, 설치 패키지, 최근 통신 시각
- **일괄 작업 (Bulk Actions)**: 여러 디바이스 선택 후 Delete / Update Package 실행
- **개별 액션**: View (상세보기) / Edit (정보 편집) / Delete (등록 해제)

**주요 플로우:**
1. 디바이스 리스트 진입 → 전체 디바이스 목록 표시
2. 필터/검색으로 원하는 디바이스 찾기
3. 개별 디바이스 클릭 또는 액션 버튼으로 작업 수행:
   - [View]: 디바이스 상세 화면으로 이동
   - [Edit]: 디바이스 정보(이름, 설명, 매장 등) 편집
   - [Delete]: 등록 해제 (확인 모달)
5. [+ Register Device] 버튼으로 신규 디바이스 등록

#### 디바이스 상세 ([wireframe/device-detail.md](./wireframe/device-detail.md))
- 리스트·검색: 디바이스ID, MAC, 태그, 제품
- 상세: 인증 상태, 설치된 패키지·콘텐츠·버전, 최근 로그
- 원격 액션: Force Update / Restart / Collect Logs
- 인증 관리: MAC 등록(최대 2개), 인증 히스토리 조회

### 8) 사용자·권한 관리
- 사용자 초대·삭제, 역할(Role) 생성·편집
- 역할별 접근 정책: 화면·액션 가시성 제어
- 감사로그(Audit): 중요 액션(배포/롤백/삭제) 기록

### 9) 모니터링·리포트
- 활동 로그 및 이벤트 스트림(필터 가능)
- 배포 리포트: 성공/실패 디바이스 수, 실패 원인 통계
- 인증·사용량 리포트: 인증 횟수 기반 사용량 분석

## 공통 컴포넌트·패턴
- 리스트뷰: 컬럼 커스터마이즈, 멀티셀렉트, Bulk action
- 디테일 패널: 슬라이드오버 또는 모달로 신속 편집
- 진행표시: 작업 ID, 진행 바, 단계별 상태 메세지
- 확인 모달: 삭제·배포·롤백 등 민감한 액션에 필수
- 알림/토스트: 성공·실패·경고 메시지, 링크(작업 상세)

## 에러·알림 처리 원칙
- 비동기 작업은 작업 ID 반환(추적용)
- 실패는 원인 카테고리화(네트워크/검증/서버)하여 복구 제안 제시
- 재시도·롤백 옵션과 예상 영향(대상 수, 다운타임) 표기

## 접근성·국제화
- 다국어 지원(한국어/영어 기본)
- 날짜·시간 로케일 표기, 숫자 포멧
- 키보드 내비게이션 및 스크린리더 호환성 확보

## 보안·감사
- 권한 기반 접근 제어(최소 권한 원칙)
- 모든 배포·롤백·삭제는 감사로그에 기록
- 민감 정보는 UI에서 마스킹 및 접근 통제

## UX 시나리오(간략)
- 신규 콘텐츠 등록 → 스테이징 배포 → 테스트 확인 → 퍼블리시 및 패키지 포함 → 페이즈드 배포
- 배포 중 심각 오류 발생 → 모니터링에서 감지 → 롤백 수행 → 롤백 리포트 확인

- 권한 기반 메뉴 맵 정의 및 주요 에러 케이스 UI 모크업
