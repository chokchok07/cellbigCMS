# Product Editor UI 와이어프레임

## 개요
- 목적: 신규 제품을 등록하거나 기존 제품 정보를 편집합니다.
- 접근 경로: 
  - 신규: Product List에서 [+ New Product] 버튼 클릭
  - 편집: Product List 또는 Product Detail에서 [Edit] 버튼 클릭

## 주요 구성 요소

### 1. 공통 사이드바
- Dashboard, Content, Package, Product, Devices 메뉴 항목
- 현재 페이지(Product) 강조 표시

### 2. 헤더
- 페이지 제목: "🏢 Product Editor"
- 부제: 
  - 신규: "Create New Product"
  - 편집: "Edit: {제품명}"
- 액션 버튼:
  - [Cancel]: 변경사항 취소하고 이전 페이지로 복귀 (확인 모달)
  - [Save]: 제품 저장

### 3. 기본 정보 섹션 (*필수 항목 표시)

#### Product ID
- **신규 등록**: 
  - 자동 생성 옵션: "product-{랜덤문자열}" 자동 입력
  - 수동 입력 옵션: 사용자가 직접 입력 (중복 검사)
  - 형식: 영문 소문자, 숫자, 하이픈만 허용
- **편집 모드**: 읽기 전용 (회색 배경, 수정 불가)

#### Product Name *필수
- 입력 필드: 텍스트 (최대 100자)
- 검증: 빈 값 불가, 중복 제품명 경고

#### Description
- 텍스트 영역: 여러 줄 입력 가능 (최대 500자)
- 선택 사항
- 플레이스홀더: "제품에 대한 상세 설명을 입력하세요..."

#### Default Package
- 드롭다운: 등록된 패키지 목록에서 선택
- 선택 사항
- 편집 모드에서는 현재 이 제품에 연결된 패키지만 표시

### 4. 연결된 패키지 섹션 (편집 시에만 표시)
- 현재 제품에 연결된 패키지 목록
- 각 패키지 정보:
  - 패키지 ID (버전)
  - 포함 콘텐츠 수
  - [Remove] 버튼: 패키지 연결 해제
- [+ Add Package] 버튼: 새 패키지 연결 (모달)

### 5. 메타데이터 섹션 (편집 시에만 표시)
- Created: 생성일시 및 생성자
- Updated: 최종 수정일시 및 수정자
- 읽기 전용 정보

### 6. 하단 액션 버튼
- **[Cancel]**: 
  - 변경사항이 있으면 확인 모달 표시
  - "변경사항을 저장하지 않고 나가시겠습니까?"
  - 확인 → 이전 페이지로 복귀
- **[Save Product]**:
  - 필수 항목 검증
  - 저장 성공 → Product Detail 페이지로 이동
  - 저장 실패 → 에러 메시지 표시

## 주요 상호작용 플로우

### 1. 신규 제품 등록
1. Product List에서 [+ New Product] 클릭
2. Product Editor 오픈 (빈 폼)
3. Product ID 자동 생성 또는 수동 입력
4. Product Name 입력 (필수)
5. Description 입력 (선택)
6. Default Package 선택 (선택)
7. [Save Product] 클릭
8. 검증 통과 → 저장 → Product Detail 페이지로 이동

### 2. 기존 제품 편집
1. Product List 또는 Product Detail에서 [Edit] 클릭
2. Product Editor 오픈 (기존 정보 로드)
3. Product ID는 읽기 전용 상태
4. Name, Description, Default Package 수정 가능
5. 연결된 패키지 섹션에서 패키지 추가/제거
6. [Save Product] 클릭
7. 검증 통과 → 저장 → Product Detail 페이지로 이동

### 3. 패키지 추가 (편집 시)
1. [+ Add Package] 버튼 클릭
2. 패키지 선택 모달 표시
3. 사용 가능한 패키지 목록 (이미 연결된 패키지 제외)
4. 패키지 선택 후 [Add] 클릭
5. 연결된 패키지 목록에 추가

### 4. 패키지 제거 (편집 시)
1. 특정 패키지의 [Remove] 클릭
2. 확인 모달: "이 패키지를 제품에서 제거하시겠습니까?"
3. Default Package인 경우 추가 경고
4. 확인 → 목록에서 제거 (저장 시 반영)

### 5. 취소 (변경사항 있을 때)
1. [Cancel] 버튼 클릭
2. 확인 모달: "변경사항을 저장하지 않고 나가시겠습니까?"
3. 확인 → 이전 페이지로 복귀
4. 취소 → 편집 계속

## 검증 규칙

### 필수 항목
- **Product Name**: 빈 값 불가
  - 에러 메시지: "제품명을 입력해주세요."

### Product ID 검증 (신규 등록 시)
- 형식: 영문 소문자, 숫자, 하이픈만 허용
- 중복 검사: 이미 존재하는 ID는 사용 불가
- 에러 메시지: 
  - "Product ID는 영문 소문자, 숫자, 하이픈만 사용할 수 있습니다."
  - "이미 사용 중인 Product ID입니다."

### 길이 제한
- Product Name: 최대 100자
- Description: 최대 500자

### Default Package 검증
- 선택된 패키지가 이 제품에 연결되어 있어야 함 (편집 시)
- 신규 등록 시에는 입력하지 않아도 됨

## 에러 처리

### 저장 실패
```
❌ Failed to save product
Error: {상세 에러 메시지}
[Retry] [Cancel]
```

### 중복 Product ID
```
⚠️ Product ID already exists
Please choose a different Product ID.
[OK]
```

### 네트워크 에러
```
❌ Network Error
Could not connect to the server. Please check your connection.
[Retry] [Cancel]
```

## UX 노트
- 자동 저장 기능은 제공하지 않음 (명시적 저장 필요)
- 변경사항이 있을 때 페이지 이탈 시 확인 모달 표시
- Product ID는 신중하게 입력 (변경 불가)
- 필수 항목은 * 표시로 명확히 구분
- Default Package는 나중에 Product Detail에서도 변경 가능
- 패키지 연결은 편집 모드에서만 가능
- 에러 메시지는 필드 바로 아래 또는 상단에 명확히 표시

## 반응형 고려사항
- 모바일: 섹션을 세로로 스택, 버튼은 풀 너비
- 태블릿: 폼 너비 조정, 레이블과 입력 필드 배치 조정
- 데스크톱: 최대 너비 제한 (읽기 편한 폼 너비)

## 접근성
- 모든 입력 필드에 레이블 연결
- 필수 항목은 aria-required="true"
- 에러 메시지는 aria-live="polite"로 스크린리더에 전달
- 키보드로 모든 기능 접근 가능 (Tab, Enter, Esc)
