# 대시보드 와이어프레임

## ASCII Layout

```
┌───────────────────────────────────────────────────────────────────────────┐
│ [Product ▼]  🔍 Search...          🔔 (3)  👤 Admin                      │
├─────────────┬─────────────────────────────────────────────────────────────┤
│             │                                                             │
│ 📊 Dashboard│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐      │
│ 📝 Content  │  │ Deployment   │ │ Active       │ │ Pending      │      │
│ 📦 Package  │  │ Summary      │ │ Devices: 120 │ │ Packages: 7  │      │
│ 🏢 Product  │  │ ✓ 95%        │ │              │ │              │      │
│ 💻 Devices  │  │ ✗ 5%         │ │ 🟢 Online    │ │ 🟡 Waiting   │      │

│             │  │                                                   │      │

│             │  Quick Actions: [+ New Content] [+ New Package] [🔍 Search]│

```

목적: 관리자/운영자가 시스템 상태를 한눈에 파악하고 빠른 작업을 시작할 수 있도록 구성.

레이아웃(상단 → 좌우):
- 상단 바: [제품 선택] [전역 검색] [알림 아이콘] [사용자 메뉴]
- 좌측 사이드바: Dashboard / Content / Package / Product / Devices / Reports / Settings
- 메인 그리드(3열 권장)
  - 카드(Deployment Summary): 최근 배포 상태(성공/실패 비율, 진행중 작업)
  - 카드(Recent Activity): 최근 작업 로그(클릭 시 상세 이동)
  - 카드(Alerts): 중요 알림 및 빠른 액션 버튼
  - 카드(Quick Actions): New Content / New Package / Search Device
  - 차트(Deployment Trend): 시간별 배포 성공률
  - 테이블(Recent Failed Deployments): 실패 항목 요약 + 상세 링크

주요 컴포넌트 및 상호작용:
- 카드 클릭 → 관련 상세 페이지로 이동
- Quick Action 클릭 → 모달/새 페이지로 이동
- 알림 클릭 → 해당 리소스 하이라이트

우선순위 액션:
- 이상 징후(에러) 확인 → 배포 모니터링 진입
- 신규 콘텐츠 업로드 → 콘텐츠 에디터 오픈

비고:
- 실시간 상태가 중요하므로 배포 카드와 로그는 페이징 대신 실시간 스트리밍/폴링 옵션 제공.
