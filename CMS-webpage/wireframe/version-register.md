# 버전 등록 팝업

## 개요
콘텐츠의 새 버전을 등록하기 위한 모달 팝업 화면입니다. 시맨틱 버저닝(major.minor.patch), 업데이트 타입(강제/선택), 배포 환경(상용/테스트), 파일 업로드, 버전 설명을 입력합니다.

## 주요 필드

### 1. 버전 (major.minor.patch)
- 세 개의 숫자 입력 필드로 구성
- 각 필드는 0-999 범위
- 자동으로 "major.minor.patch" 형식 생성

### 2. 업데이트 타입
- **강제 업데이트**: 디바이스가 반드시 업데이트해야 함
- **선택 업데이트**: 디바이스가 선택적으로 업데이트 (기본값)

### 3. 배포 환경
- **상용**: 프로덕션 환경 배포
- **테스트**: 테스트 환경 배포 (기본값)

### 4. 버전 파일
- ZIP 아티팩트 파일 업로드
- 파일 선택 후 자동으로 체크섬(SHA-256) 계산
- 파일 크기 표시

### 5. 파일 확인
- 업로드된 파일의 무결성 검증
- 녹색 버튼으로 확인 완료 표시

### 6. 버전 설명
- 변경 사항 및 배포 노트 작성
- 다중 라인 텍스트 입력

## 동작 흐름
1. 버전 번호 입력 (major, minor, patch)
2. 업데이트 타입 선택 (강제/선택)
3. 배포 환경 선택 (상용/테스트)
4. ZIP 파일 업로드
5. 파일 확인 버튼 클릭하여 무결성 검증
6. 버전 설명 작성
7. 등록 버튼 클릭하여 서버에 전송

## API 연동
- `POST /contents/{contentId}/versions`
- 요청 본문:
  ```json
  {
    "version_major": 2,
    "version_minor": 1,
    "version_patch": 0,
    "update_type": "optional",
    "deployment_target": "test",
    "artifact": "(file)",
    "release_notes": "버전 설명 내용"
  }
  ```
