## 목적
관리자가 시스템에 등록된 모든 콘텐츠를 한눈에 확인하고, 검색·필터링·정렬·일괄 작업을 수행할 수 있는 콘텐츠 관리의 중심 화면입니다.

## 레이아웃 구조

### 상단 바
- 제품 선택 드롭다운: 특정 제품의 콘텐츠만 필터링
- 전역 검색: 시스템 전체 검색
- 알림 아이콘: 배포/업데이트 알림 표시
- 사용자 메뉴: 프로필, 설정, 로그아웃

### 좌측 사이드바
- 대시보드, 콘텐츠, 패키지, 제품, 디바이스, 리포트, 설정 메뉴
- 현재 페이지: 콘텐츠 (📝 Content) 활성화 상태

### 메인 영역

#### 1) 헤더
- 페이지 제목: "📝 Content Library"
- [+ New Content] 버튼: 신규 콘텐츠 등록 화면으로 이동

#### 2) 필터 패널
- **검색**: 키워드로 제목, 설명 검색
- **Type**: All / single / container / child / mixed
- **Status**: All / Draft / Staged / Published / Deprecated
- **Product**: All / 제품 목록 드롭다운
- **Category**: All / 카테고리 목록 (게임, 교육, 엔터테인먼트 등)
- **[Apply Filters]** 버튼: 필터 조건 적용

#### 3) 일괄 작업 (Bulk Actions)
- **☑ Select All**: 현재 페이지의 모든 항목 선택
- **Bulk Actions 드롭다운**: 
  - Publish (선택된 항목 일괄 퍼블리시)
  - Archive (아카이브)
  - Delete (일괄 삭제 - 확인 모달 필요)
- **[Apply]** 버튼: 선택된 작업 실행

#### 4) 콘텐츠 리스트 테이블
컬럼 구성:
- **☑ (체크박스)**: 개별 선택
- **📄 Title**: 
  - 썸네일 이미지 [📷]
  - 콘텐츠 제목 (다국어 지원 - 현재 로케일 표시)
  - Category 표시
- **Type**: single / container / child / mixed
- **Status**: 
  - Published 🟢
  - Staged 🟡
  - Draft ⚪
  - Deprecated ⚫
- **Version**: 최신 버전 번호 (예: v2.1)
- **Actions**:
  - [Edit]: content-editor 화면으로 이동 (해당 콘텐츠 편집)
  - [Del]: 삭제 (확인 모달)
  - [Ver]: 버전 히스토리 및 버전 등록 화면으로 이동

#### 5) 페이지네이션
- "Showing 1-4 of 47": 현재 표시 범위 및 전체 개수
- [◀ Previous] [1] [2] [3] ... [12] [Next ▶]: 페이지 내비게이션

## 주요 상호작용 플로우

### 1. 콘텐츠 검색 및 필터링
1. 검색어 입력 또는 필터 조건 선택
2. [Apply Filters] 클릭
3. 리스트가 조건에 맞게 갱신됨
4. URL 쿼리 파라미터 업데이트 (북마크/공유 가능)

### 2. 신규 콘텐츠 등록
1. [+ New Content] 버튼 클릭
2. 콘텐츠 에디터 화면([content-editor](./content-editor.md))으로 이동
3. 메타데이터 입력 및 저장
4. 리스트로 복귀 (새 항목 추가됨)

### 3. 콘텐츠 편집
1. 특정 콘텐츠의 [Edit] 버튼 클릭
2. content-editor 화면([content-editor](./content-editor.md))으로 이동 (해당 contentId 로드)
3. 수정 후 저장
4. 리스트로 복귀 (변경사항 반영)

### 4. 버전 관리
1. 특정 콘텐츠의 [Ver] 버튼 클릭
2. 버전 등록 화면([version-register](./version-register.md))으로 이동
3. 버전 히스토리 확인 및 신규 버전 업로드
4. 리스트로 복귀

### 5. 일괄 작업 (Bulk Actions)
1. 여러 콘텐츠 체크박스 선택 (또는 Select All)
2. Bulk Actions 드롭다운에서 작업 선택 (Publish / Archive / Delete)
3. [Apply] 클릭
4. 확인 모달 표시 (특히 삭제 시)
5. 확인 후 작업 실행
6. 성공/실패 토스트 메시지 표시
7. 리스트 자동 갱신

### 6. 콘텐츠 삭제
1. 특정 콘텐츠의 [Del] 버튼 클릭
2. 확인 모달: "정말 삭제하시겠습니까? (연관된 패키지에 영향이 있을 수 있습니다)"
3. 확인 → 삭제 수행
4. 성공 토스트 및 리스트 갱신

### 7. 정렬 (추가 기능)
- 컬럼 헤더 클릭 → 해당 컬럼 기준으로 오름차순/내림차순 정렬
- 예: Title 클릭 → 알파벳순 정렬, Version 클릭 → 버전 번호순 정렬

## 컴포넌트 및 검증

### 필터 패널
- 필터 조건은 URL 쿼리 파라미터로 관리 (북마크/공유 지원)
- 초기화 버튼 제공: 모든 필터를 기본값으로 리셋

### 체크박스 및 일괄 작업
- Select All 체크박스: 현재 페이지의 모든 항목 선택/해제
- 개별 체크박스: 각 항목 선택
- 선택된 항목 개수 표시 (예: "3 items selected")
- Bulk Actions는 최소 1개 이상 선택되어야 활성화

### 상태 표시
- Published 🟢: 프로덕션 배포 가능 상태
- Staged 🟡: 테스트 배포 중 또는 스테이징 단계
- Draft ⚪: 작업 중, 미완성 상태
- Deprecated ⚫: 더 이상 사용하지 않음, 레거시

### 썸네일 이미지
- Thumbnail이 없는 경우 기본 아이콘 [📄] 표시
- Thumbnail이 있는 경우 작은 이미지 미리보기 [📷] 표시
- 클릭 시 큰 이미지 미리보기 모달 (선택적 기능)

### 페이지네이션
- 기본 페이지 크기: 10개 (설정에서 변경 가능)
- 페이지 번호 클릭 시 해당 페이지로 이동
- Previous/Next 버튼으로 앞뒤 페이지 이동

## 에러 및 엣지 케이스 처리

### 콘텐츠 없음
- 필터 조건에 맞는 콘텐츠가 없을 경우:
  ```
  📭 No content found
  Try adjusting your filters or [Create New Content]
  ```

### 삭제 시 의존성 경고
- 콘텐츠가 패키지에 포함되어 있을 경우:
  ```
  ⚠️ Warning: This content is included in 3 packages.
  Deleting it may affect deployments.
  [View Packages] [Cancel] [Delete Anyway]
  ```

### 일괄 작업 실패
- 일부 항목 실패 시:
  ```
  ✓ 5 items processed successfully
  ✗ 2 items failed (insufficient permissions)
  [View Details]
  ```

## UX 노트
- 콘텐츠 리스트는 콘텐츠 관리의 중심 화면입니다
- 검색·필터링·정렬 기능으로 원하는 콘텐츠를 빠르게 찾을 수 있어야 합니다
- 일괄 작업(Bulk Actions)으로 여러 콘텐츠를 효율적으로 관리할 수 있습니다
- 상태별 시각적 구분(색상 코드)으로 콘텐츠 상태를 직관적으로 파악할 수 있습니다
- 썸네일 이미지로 콘텐츠를 시각적으로 식별할 수 있습니다
- 페이지네이션으로 대량의 콘텐츠를 효율적으로 탐색할 수 있습니다
- 각 콘텐츠에서 바로 편집/삭제/버전 관리 작업으로 이동할 수 있어 작업 흐름이 원활합니다

## 반응형 고려사항
- 모바일: 테이블을 카드 뷰로 전환, 필터는 접을 수 있는 패널로 변경
- 태블릿: 사이드바를 토글 가능하도록 변경, 컬럼 간격 조정
- 데스크톱: 전체 테이블 뷰, 모든 기능 표시

## 접근성
- 키보드 내비게이션 지원: Tab, Enter, Space로 모든 기능 접근 가능
- 스크린리더 지원: 각 컬럼, 버튼, 상태에 적절한 레이블 제공
- 색상만으로 상태를 구분하지 않음: 아이콘(🟢🟡⚪⚫) + 텍스트 병행
