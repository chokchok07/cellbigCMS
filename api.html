1. 개요 및 공통 사항

본 문서는 Cellbig CMS 시스템의 서버 API 명세서입니다. 기획서 및 와이어프레임을 기반으로 작성되었습니다.

기본 정보

  • Base URL: https://api.cms.cellbig.com/api/v1 (예시)
  • Content-Type: application/json
  • Encoding: UTF-8

인증 방식 (Authentication)

  • Admin/Manager: Bearer Token (JWT) 헤더 사용 (Authorization: Bearer <token>)
  • Device Client:
    • 최초 등록 전: MAC Address 기반 등록 요청
    • 등록 후: 발급된 x-device-token 또는 JWT 사용

공통 응답 포맷 (표준)

{ "success": true, // 성공 여부 (boolean) "data": { ... }, // 실제 데이터 Payload "error": null // 에러 발생 시 에러 객체 (code, message) }

2. Device / Client API

콘텐츠 플레이어(클라이언트)가 사용하는 API입니다. 디바이스 등록, 인증, 업데이트 확인 등을 수행합니다.

POST /devices/register Client 디바이스 최초 등록 요청

클라이언트 설치 후 최초 실행 시, 서버에 디바이스를 등록합니다.

Parameters (Body)

FieldTypeRequiredDescription
macAddressstringYesPC의 물리적 MAC 주소 (식별자)
systemInfoobjectNoOS 버전, CPU, RAM 등 시스템 정보
clientVersionstringYes클라이언트 어플리케이션 버전

Response (200 OK)

{ "success": true, "data": { "deviceId": "dev_1234567890", "status": "PENDING_APPROVAL", // 관리자 승인 대기 "accessToken": "eyJhbG..." // 제한적 권한 토큰 } }
PUT /devices/{deviceId}/info Client 디바이스 정보 정기 갱신 (월/분기별)

디바이스의 하드웨어 스펙, OS 정보, 최신 클라이언트 버전 등을 정기적(예: 월/분기 1회)으로 서버에 동기화합니다.

Parameters (Body)

FieldTypeRequiredDescription
metaobjectYesCPU, RAM, GPU, 디스크, OS, 해상도 등 최신 하드웨어 사양 스펙 JSON
clientVersionstringYes클라이언트 어플리케이션 현재 버전
networkInfoobjectNo현재 로컬 IP 등 네트워크 정보
reportPeriodstringNoQUARTERLY, MONTHLY 등 보고 주기 명시

Response (200 OK)

{ "success": true, "data": { "updatedAt": "2026-03-11T10:00:00Z", "message": "디바이스 정보가 성공적으로 갱신되었습니다." } }
POST /devices/certify Client 콘텐츠 실행 인증 (상시/One-time)

콘텐츠 실행 시 유효한 디바이스 및 라이선스인지 확인합니다.

Parameters (Body)

FieldTypeRequiredDescription
deviceIdstringYes디바이스 ID
contentIdstringYes실행하려는 콘텐츠 ID
macAddressstringYes보안 검증용 MAC 주소
authTypestringYes'ALWAYS' | 'ONETIME'

Response (200 OK)

{ "success": true, "data": { "authorized": true, "sessionId": "sess_abc123", // 로그 전송 시 사용 "license": { "validUntil": "2025-12-31T23:59:59Z", "type": "SUBSCRIPTION" } } }
GET /devices/{deviceId}/packages/check-update Client 업데이트 확인

현재 디바이스에 할당된 패키지의 최신 버전을 확인합니다.

Parameters (Query)

FieldTypeRequiredDescription
currentPackageVersionstringYes현재 설치된 패키지 버전

Response (200 OK)

{ "success": true, "data": { "hasUpdate": true, "targetPackage": { "packageId": "pkg_sandcraft_full", "version": "1.5.0", "releaseNote": "신규 콘텐츠 2종 추가", "forceUpdate": false }, "contents": [ // 업데이트가 필요한 콘텐츠 목록만 전달하거나 전체 목록 전달 { "contentId": "cont_24", "version": "1.2.0", "action_type": "UPDATE", "target_entity": "Unknown", // UPDATE, CREATE, DELETE, KEEP "checksum": "sha256_hash_value..." } ] } }
GET /contents/{contentId}/versions/{version}/download Client 콘텐츠 다운로드 URL 요청

특정 버전의 콘텐츠 바이너리(ZIP) 다운로드 URL(Pre-signed URL 등)을 요청합니다.

Response (200 OK)

{ "success": true, "data": { "downloadUrl": "https://cdn.cellbig.com/contents/app_v1.2.0.zip?token=...", "expiresAt": "2026-03-10T12:00:00Z", "fileSize": 104857600, "checksum": "a1b2c3d4..." } }

3. Admin Management API

CMS 웹 관리자 페이지에서 사용하는 API입니다.

인증 (Authentication)

POST /auth/login Admin 관리자 로그인

Parameters

username, password

Response

JWT Access Token, Refresh Token 반환

POST /auth/register Admin 운영자/관리자 회원가입

신규 운영자 계정을 생성 요청합니다. (생성 후 Super Admin 승인 필요)

Parameters (Body)

FieldTypeRequiredDescription
login_idstringYes로그인 아이디
passwordstringYes암호화된 계정 비밀번호
user_namestringYes유저 이름
emailstringYes연락처 이메일
descriptionstringNo사용자 계정 정보 (설명)

Response (200 OK)

{ "success": true, "data": { "userId": "user_pending_123", "status": "PENDING_APPROVAL", "message": "계정이 생성되었습니다. 관리자 승인 후 로그인 가능합니다." } }
GET /auth/me Admin 내 정보 및 권한 조회

현재 로그인된 사용자의 상세 정보와 권한(Role)을 조회합니다. 프론트엔드 메뉴 구성 시 사용됩니다.

{ "success": true, "data": { "userId": "user_uuid_123", "username": "admin01", "user_name": "시스템 관리자", "email": "admin@example.com", "description": "최고 관리자 계정입니다.", "role": "super_admin", // super_admin, local_manager, store_manager "scopeIds": [], // 관리 권한이 있는 지역/스토어 ID 목록 (전체 관리자는 빈 배열) "permissions": ["device:read", "device:write", "user:manage"] } }
PUT /auth/me Admin 내 정보 수정

현재 로그인된 사용자의 개인정보(비밀번호, 이름 등)를 수정합니다.

Parameters

FieldTypeRequiredDescription
currentPasswordstringNo현재 비밀번호 (비밀번호 변경 시 필수)
newPasswordstringNo새 비밀번호
user_namestringNo유저 이름 변경 시
emailstringNo연락처 이메일 변경 시
descriptionstringNo사용자 계정 정보 (설명) 변경 시

사용자 및 권한 관리 (Users & Permissions)

GET /users/me Admin 내 계정 정보 조회

현재 로그인된 사용자의 상세 정보를 조회합니다. (본인 계정 수정 시 필요한 데이터 로드용)

PUT /users/me Admin 내 계정 정보 수정

현재 로그인된 사용자의 이름, 이메일 등의 기본 정보를 수정합니다.

Parameters (Body)

FieldTypeRequiredDescription
user_namestringYes변경할 유저 이름
emailstringYes변경할 연락처 이메일
PUT /users/me/password Admin 내 비밀번호 변경

현재 비밀번호를 확인하고 새로운 비밀번호로 변경합니다.

Parameters (Body)

FieldTypeRequiredDescription
currentPasswordstringYes현재 비밀번호
newPasswordstringYes새 비밀번호
POST /users/signup Admin 운영자 계정 가입 요청

새로운 관리자/운영자 계정 가입을 요청합니다. 기본적으로 PENDING(대기) 상태로 생성됩니다.

GET /users/pending Admin 승인 대기 계정 목록 조회

가입 후 아직 관리자 승인을 받지 못한 PENDING 상태의 사용자 목록을 조회합니다. (Account Approvals 화면용)

필요 권한: super_admin

POST /users/{userId}/approve Admin 계정 가입 승인

대기 중인 사용자의 가입을 승인하여 ACTIVE 상태로 변경하고 Role을 부여합니다.

POST /users/{userId}/reject Admin 계정 가입 거절

대기 중인 사용자의 가입 요청을 거절합니다. (해당 정보 삭제 또는 REJECTED 상태 처리)

GET /users Admin 사용자 목록 조회

시스템에 등록된 전체 운영자(관리자) 계정 목록을 조회합니다.

필요 권한: super_admin (최고 관리자 전용)

{ "success": true, "data": [ { "user_id": "u1", "login_id": "manager_seoul", "user_name": "서울 관리자", "email": "manager_seoul@example.com", "description": "서울 지역 총괄 관리자 계정", "role": "Local", "scopeIds": ["loc_seoul"], "status": "ACTIVE", "lastLoginAt": "2026-03-09T10:00:00Z" } ] }
POST /users Admin 새 운영자 계정 생성

Parameters (Body)

FieldTypeRequiredDescription
login_idstringYes로그인 아이디
passwordstringYes암호화된 초기 비밀번호
user_namestringYes유저 이름
emailstringYes연락처 이메일
descriptionstringNo사용자 계정 정보 (설명)
roleenumYesSuper / Operation / Local / Technician
scopeIdsarrayNo관리 대상 ID 목록 (지역ID 또는 스토어ID)
PUT /users/{userId} Admin 운영자 정보 및 권한 수정

사용자 정보, 역할, 관리 범위(scopeIds)를 수정하거나 계정 상태(정지/활성)를 변경합니다. (최고/중간 관리자용)

필요 권한: `super_admin` 또는 `operation_manager`

Parameters (Body)

FieldTypeRequiredDescription
user_namestringNo변경할 유저 이름
emailstringNo변경할 연락처 이메일
passwordstringNo변경할 비밀번호
descriptionstringNo사용자 계정 정보 (설명) 수정
roleenumNo역할 권한 (Super / Operation / Local / Technician)
scopeIdsarrayNo관리할 지역(LocalArea) 또는 스토어 ID 목록
statusenumNo계정 상태 (Active, Suspended)

Response (200 OK)

{ "success": true, "data": { "userId": "user_123", "role": "local_manager", "scopeIds": ["store_busan_01"], "updatedAt": "2026-03-11T13:00:00Z" } }
PATCH /{entity}/{id}/status Admin 공통 상태 토글 (활성/비활성)

리스트 페이지의 Active/Inactive 버튼이나 모달 등을 통해 사용되는 가벼운 상태 변경 공통 API입니다.
entity 경로에는 users, products, packages, contents, devices 등이 공통 규격으로 들어갈 수 있습니다.

Parameters (Body)

FieldTypeRequiredDescription
statusstringYesACTIVE 또는 INACTIVE

제품 (Products)

GET /products Admin 제품 목록 조회

등록된 모든 제품(솔루션) 목록을 조회합니다.

POST /products Admin 제품 생성

패키지 (Packages)

GET /products/{productId}/packages Admin 제품 하위 패키지 목록
PUT /packages/{packageId} Admin 패키지 정보 수정 / 콘텐츠 매핑

패키지에 포함될 콘텐츠 ID 목록을 업데이트합니다.

{ "name": "샌드크래프트 프리미엄 패키지", "contentIds": ["cont_1", "cont_2", "cont_5"] }

콘텐츠 (Contents)

POST /contents Admin 콘텐츠 메타데이터 생성

콘텐츠의 기본 정보(이름, 타입, 썸네일 등)를 생성합니다.

POST /contents/{contentId}/versions Admin 새 버전 업로드 (파일 포함)

Multipart/form-data 요청으로 콘텐츠 바이너리 파일(.zip)과 버전 정보를 업로드합니다.

Parameters (Multipart)

  • file: 바이너리 파일
  • versionMajor, versionMinor, versionPatch
  • releaseNotes

지역 및 스토어 (LocalArea & Store)

GET /local-areas Admin 지역 목록 (트리 구조)
POST /stores Admin 스토어 생성

특정 LocalArea 하위에 스토어를 생성합니다.

디바이스 관리 (Device Management)

GET /admin/devices Admin 전체 디바이스 목록 (검색/필터)
PUT /admin/devices/{deviceId}/approve Admin 디바이스 승인 및 스토어 할당

승인 대기 상태인 디바이스를 승인하고, 특정 스토어 및 패키지를 할당합니다.

{ "storeId": "store_gangnam_01", "packageId": "pkg_sand_premium", "status": "ACTIVE" }

라이선스 (License)

POST /licenses Admin 라이선스(시리얼키) 발급
{ "productId": "prod_sandcraft", "type": "SUBSCRIPTION", "durationDays": 365, "count": 10 // 한 번에 10개 생성 }

시스템 업데이트 및 패치노트 (Notices)

GET /system/notices Admin 업데이트 및 패치노트 목록 조회 (조회 권한: 전체 관리자)

POST /system/notices Super Admin 신규 패치노트 등록 (생성 권한: Super Admin 전용)

NameTypeRequiredDescription

Response (201 Created)

{ "success": true, "data": { "id": "not-002", "status": "CREATED" } }

Data Models

This section outlines the primary data structures used within the CellbigCMS API.

MODEL Device

Description

Represents a physical device that connects to the CMS.

Properties

Property Type Description
deviceId String Unique identifier for the device (e.g., dev-001)
name String Human-readable name for the device
macAddresses Array[String] List of MAC addresses associated with the device's network interfaces
status String Current activation status (e.g., ACTIVE, INACTIVE)
MODEL License

Description

Represents a license granted to a device for a specific product.

Properties

Property Type Description
licenseKey String The serial key or token string
type String Type of license (e.g., PERPETUAL, SUBSCRIPTION)
expireDate Date Expiration date for subscription licenses (null for perpetual)
MODEL Notice

Description

Represents a system update log or patch note displayed on the dashboard.

Properties

Property Type Description
id String Unique identifier for the notice (e.g., not-001)
title String Headline of the update (e.g., v1.2.0 업데이트)
content String Detailed HTML/Markdown content explaining the changes
author String The admin who created the notice
createdAt Date Timestamp when the patch note was published
