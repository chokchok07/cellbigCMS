# CMS 관리 웹페이지 — 주요 기능

## 개요
이 문서는 CMS 관리(관리자) 웹페이지가 제공해야 하는 핵심 기능을 간결하게 정리합니다. 각 항목은 관리자 관점의 주요 요구사항과 기대 동작을 설명합니다.

## 핵심 기능

- **대시보드**: 시스템 요약(활성 디바이스, 오류 알림, 최근 활동)을 한눈에 표시.
- **권한/역할 관리**: 관리자·운영자·읽기전용 등 역할 기반 접근제어와 권한 세부 설정.
- **사용자 계정 관리**: 계정 생성·편집·비활성화, 비밀번호 초기화.
- **콘텐츠(Asset) 관리**: 콘텐츠 업로드, 버전 관리, 메타데이터 편집(제목·설명·태그 등), 미리보기 제공.
- **패키지(콘텐츠 묶음) 관리**: 패키지 생성·편집, 포함 콘텐츠 선택, 게시 상태 관리.
- **디바이스 등록 및 그룹 관리**: 디바이스 등록/삭제, 디바이스 그룹(태그) 생성, 그룹 단위 설정 적용.
- **인증 및 보안 설정**: 콘텐츠 인증 방식(상시/일회성), 인증키/토큰 관리, 인증 로그 조회.
- **모니터링·상태 확인**: 디바이스 상태(온라인/오프라인), 최신 접속 시간, 콘텐츠 실행 상태 실시간 조회.
- **로그·이벤트 뷰어**: 시스템·디바이스 관련 로그 검색, 필터(기간·레벨·디바이스) 및 로그 다운로드.
- **통계·리포트**: 오류 통계, 콘텐츠 사용률, 디바이스 통계 대시보드 및 CSV/PDF 추출.
- **알림·경보**: 이메일/SMS/웹훅 알림 설정, 임계값 기반 경보(오류율, 디바이스 오프라인 비율 등).
- **설정·시스템 관리**: 시스템 설정(네트워크, 저장소, 백업), API 키 관리, 로깅/감사 설정.
- **API 관리 및 통합**: 외부 시스템 연동을 위한 API 키 발급, 사용량 모니터링, 문서 링크 제공.
- **테스트·검증 도구**: 콘텐츠 무결성 검사, 샌드박스 배포(테스트 디바이스), 시뮬레이션 로그 확인.
- **다국어·지역화 지원**: 관리자 UI 다국어 설정, 콘텐츠별 지역화(로케일) 관리.
- **UI/작업 편의성**: 대량 작업(배치 편집), 드래그앤드롭 업로드, 변경 히스토리(감사 로그) 제공.

## 운영 시나리오별 요구사항 요약

- **신규 콘텐츠 추가**: 콘텐츠 업로드 → 패키지에 포함 → 검증 → 게시 → 모니터링.
- **문제 발생 대응**: 오류 알림 수신 → 로그·디바이스 상태 확인 → 콘텐츠 수정 또는 패키지 변경 → 보고서 생성.
- **콘텐츠 업데이트**: 수정된 콘텐츠 업로드 → 패키지 구성 확인 → 게시 → 디바이스에서 자동 갱신.

## 참고 및 확장 포인트

- **권한 세분화**: 리소스 수준 권한(콘텐츠·패키지·디바이스 단위) 확장 고려.
- **감사·컴플라이언스**: 모든 관리자 활동의 변경 이력과 증적 보관 정책 필요.
- **성능·확장성**: 대규모 디바이스·콘텐츠 증가를 대비한 캐시·큐잉·비동기 처리 설계.
