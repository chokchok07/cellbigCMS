# 인프라(Infrastructure) 관리 기획 가이드

## 1. 개요
본 문서는 CMS 시스템 내 물리적 인프라 자원인 `LocalArea`(지역), `Store`(매장), `Device`(기기)의 관리 기능을 기획하기 위한 항목들을 정의합니다.
인프라 관리는 콘텐츠 배포의 물리적 대상(Target)을 정의하고, 현장 장비의 상태를 모니터링하며, 원격 제어를 수행하는 기반이 됩니다.

## 2. 인프라 계층 구조 (Hierarchy)
인프라는 다음과 같은 논리적/물리적 계층 구조를 가집니다.

```mermaid
graph TD
    Root[CMS Root] --> LA[LocalArea (지역/그룹)]
    LA --> ST[Store (매장/지점)]
    ST --> DV[Device (기기/PC)]
```

- **LocalArea**: 최상위 지리적/논리적 그룹 (예: 국가, 도시, 본부)
- **Store**: 실제 기기가 설치된 물리적 장소 (예: 강남점, 쇼룸, 학교)
- **Device**: 콘텐츠가 재생되는 단말기 (예: 키오스크, PC, 태블릿)

---

## 3. 기획 시 고려해야 할 주요 항목

### 3.1 LocalArea (지역 관리)
*지역 단위의 마케팅 정책이나 관리 권한을 위임하기 위한 단위*

- **데이터 속성 정의**
  - 지역명, 코드(ISO 국가코드 등), 담당자 정보
  - 상위 지역 (계층 구조 지원 여부)
- **기능 요건**
  - 지역별 콘텐츠 배포 정책 설정 (예: 한국 지역엔 한국어 콘텐츠 자동 할당)
  - 통계 집계의 기준점 (지역별 가동률, 점유율)

### 3.2 Store (매장 관리)
*실제 운영이 이루어지는 현장 단위*

- **데이터 속성 정의**
  - 매장명, 주소(Zip, Address), 위도/경도(Map 연동용)
  - 운영 시간, 휴무일 (기기 자동 On/Off 스케줄링 연동)
  - 현장 담당자 연락처 (장애 발생 시 알림 대상)
- **기능 요건**
  - **지도 연동**: 구글맵/네이버맵 API를 활용한 매장 위치 시각화
  - **매장 라이선스**: 매장 단위 과금 모델 적용 시 라이선스 키 관리
  - **그룹핑**: 여러 매장을 묶어서 관리(예: 직영점 vs 가맹점)

### 3.3 Device (기기 관리)
*가장 핵심적인 관리 대상이자 모니터링 엔드포인트*

- **데이터 속성 정의**
  - **식별자**: Device ID, MAC Address (Primary/Secondary), Serial Number
  - **하드웨어 사양**: CPU, RAM, Disk Space, OS Version (배포 적합성 판단용)
  - **상태 정보**: Heartbeat(최종 통신 시간), IP Address (내부/외부), App Version
  - **연결 장치**: 연결된 센서, 프로젝터, 터치스크린 정보
- **기능 요건**
  - **등록 프로세스 (Provisioning)**
    - 관리자 수동 등록 vs 설치 시 자동 등록(Token 인증)
    - 승인 절차 (Allow-list 기반)
  - **원격 제어 (Remote Control)**
    - 전원 관리: 재시작(Reboot), 종료(Shutdown), WOL(Wake-on-LAN)
    - 어플리케이션: 앱 재시작, 로그 파일 업로드 요청, 스크린샷 캡쳐 요청
    - 업데이트: 강제 버전 업데이트, 패치 적용
  - **모니터링 (Monitoring)**
    - **Health Check**: 주기적(예: 1분) Heartbeat 수신 및 Online/Offline 상태 판별
    - **Alert**: 오프라인 지속, 리소스 부족(Disk Full), 앱 크래시 발생 시 알림
  - **인증 관리 (Authentication Management)**
    - **하이브리드 인증 (Hybrid Auth)**: 온라인/오프라인 환경을 모두 지원하는 이원화된 인증 체계
    - **온라인 인증**: 실시간 서버 통신을 통한 토큰(Token) 기반 자동 인증 및 갱신
    - **오프라인 인증 (Key Issuance)**
      - **라이선스 키 발급**: 인터넷 연결이 불가능한 폐쇄망 환경을 위해, 관리자 페이지에서 별도의 '인증 키(License Key)' 발급 기능 제공
      - **키 생성 로직**: Device ID + MAC Address + 유효기간(Expiry)을 조합하여 위변조 불가능한 암호화 키 생성
      - **키 적용 프로세스**: 관리자가 발급받은 키 파일(또는 텍스트)을 USB 등을 통해 기기에 직접 입력하여 활성화

## 4. UI/UX 기획 포인트

### 4.1 리스트 및 조회 (List View)
- **트리 뷰 (Tree View)**: 좌측 사이드바에서 `LocalArea > Store > Device` 계층을 탐색하는 트리 구조 제공
- **상태 필터링**: Online/Offline/Warning 상태별 퀵 필터 제공
- **전체 현황판**: 지도(Map) 기반으로 매장 위치와 상태(초록/빨강 마커)를 한눈에 표시

### 4.2 상세 화면 (Detail View)
- **Device Dashboard**: 개별 기기의 CPU/Memory 차트, 네트워크 트래픽, 최근 로그 스트림 표시
- **원격 제어 패널**: 즉시 실행 가능한 명령 버튼(재시작, 원격접속 등) 배치
- **배포 이력**: 해당 기기에 언제 어떤 콘텐츠/패키지가 배포되었는지 타임라인 제공

### 4.3 모바일/현장 지원
- 현장 설치 기사를 위한 모바일 뷰 혹은 전용 앱 기획 고려
- QR 코드를 통한 기기 간편 등록 기능

---

## 5. 데이터 및 시스템 연동 (Integration)
- **인증 (Authentication)**: 기기-서버 간 보안 통신 (TLS, HMAC, Client Certificate)
- **로그 수집 (Log Aggregation)**: 기기 로컬 로그를 서버(ELK Stack 등)로 전송하는 주기 및 정책
- **알림 (Notification)**: Slack, Email, SMS 연동 (장애 발생 시)

---

## 6. 개발 우선순위 (Roadmap)
1. **Phase 1 (기반 구축)**: CRUD, 단순 Heartbeat(On/Off), 기기 등록
2. **Phase 2 (제어 및 배포)**: 원격 재시작, 콘텐츠 강제 배포, 로그 수집
3. **Phase 3 (지능형 관리)**: 지도 대시보드, 통계 리포트, 장애 예측 알림
