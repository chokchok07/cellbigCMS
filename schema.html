1. Entity Relationship Diagram (ERD)

CellbigCMS의 핵심 테이블 간의 관계도입니다. 사용자는 역할(Role)에 따라 스토어나 지역 권한(Scope)을 할당받습니다.

erDiagram tb_user ||--o{ tb_user_scope : "has access to" tb_local_area ||--o{ tb_store : "contains" tb_local_area ||--o{ tb_user_scope : "granted to (optional)" tb_store ||--o{ tb_device : "operates" tb_store ||--o{ tb_user_scope : "granted to (optional)" tb_product ||--o{ tb_package : "defines" tb_product ||--o{ tb_content : "owns tb_package ||--o{ tb_package_content : "composition" tb_content ||--o{ tb_package_content : "included in" tb_package ||--o{ tb_license : "issues" tb_license |o--o| tb_device : "activated on"

2. Data Tables Dictionary

👤 tb_user (사용자/관리자)

CMS에 접속하는 관리자 및 운영자 계정 정보 (Settings 회원가입 처리용)

ColumnTypeKey/NullDescription
user_idVARCHAR(50)PK사용자 고유 식별자
login_idVARCHAR(100)UNIQUE로그인 아이디
password_hashVARCHAR(255)NOT NULL암호화된 비밀번호
user_nameVARCHAR(100)NOT NULL사용자 성명
emailVARCHAR(150)연락처 이메일
roleENUMNOT NULLSUPER, OPERATION, LOCAL, TECHNICIAN
statusENUMNOT NULLPENDING, ACTIVE, SUSPENDED, REJECTED
created_atDATETIME생성 일시

🏢 tb_local_area (지역)

스토어들을 그룹화하는 상위 지역 개념

ColumnTypeKey/NullDescription
area_idVARCHAR(50)PK지역 고유 식별자
area_nameVARCHAR(100)NOT NULL지역 명칭 (예: 서울, 부산)
statusENUMNOT NULLACTIVE, INACTIVE

🏪 tb_store (매장/스토어)

실제 디바이스가 설치되고 운영되는 매장

ColumnTypeKey/NullDescription
store_idVARCHAR(50)PK스토어 고유 식별자
🔗 area_idVARCHAR(50)FK소속 지역 ID
store_nameVARCHAR(100)NOT NULL스토어 명칭 (예: 롯데월드타워점)
addressVARCHAR(255)상세 주소
statusENUMNOT NULLACTIVE, INACTIVE

🖥️ tb_device (디바이스)

매장에 설치되어 콘텐츠를 실행하는 PC/단말기

ColumnTypeKey/NullDescription
device_idVARCHAR(50)PK디바이스 고유 식별자
🔗 store_idVARCHAR(50)FK설치된 스토어 ID
mac_addressVARCHAR(50)UNIQUE단말기 MAC 주소 (상시인증 기준)
device_nameVARCHAR(100)디바이스 별칭
statusENUMNOT NULLACTIVE, INACTIVE
last_ping_atDATETIME마지막 서버 통신(생존) 시간

📦 tb_product (제품)

판매되는 최상위 단위 (소프트웨어 묶음군)

ColumnTypeKey/NullDescription
product_idVARCHAR(50)PK제품 고유 식별자
product_nameVARCHAR(100)NOT NULL제품 명칭
descriptionTEXT제품 설명
statusENUMNOT NULLACTIVE, INACTIVE

🗂️ tb_package (패키지)

제품의 하위 개념으로, 특정 테마별로 묶인 형태 (실제 기기가 다운로드 받는 단위)

ColumnTypeKey/NullDescription
package_idVARCHAR(50)PK패키지 고유 식별자
🔗 product_idVARCHAR(50)FK소속 제품 ID
package_nameVARCHAR(100)NOT NULL패키지 명칭
versionVARCHAR(20)현재 배포 버전 (예: 1.0.2)
statusENUMNOT NULLACTIVE, INACTIVE

🎬 tb_content (콘텐츠)

패키지를 구성하는 단일 실행파일/리소스

ColumnTypeKey/NullDescription
content_idVARCHAR(50)PK콘텐츠 고유 식별자
🔗 product_idVARCHAR(50)FK연결할 제품 ID
titleVARCHAR(100)NOT NULL콘텐츠 제목
file_urlVARCHAR(255)NOT NULL다운로드 대상 S3/CDN 경로
file_sizeBIGINT파일 용량 (Byte)
versionVARCHAR(20)개별 콘텐츠 업데이트 버전
statusENUMNOT NULLACTIVE, INACTIVE

🔗 tb_package_content (매핑 테이블)

패키지와 콘텐츠를 N:M 다대다 매핑

ColumnTypeKey/NullDescription
🔗 package_idVARCHAR(50)PK, FK연결된 패키지 ID
🔗 content_idVARCHAR(50)PK, FK연결된 콘텐츠 ID
sort_orderINT패키지 내 표시/실행 순서

🔑 tb_license (라이선스/시리얼)

기기가 패키지를 사용하기 위한 인증 키 (Serial)

ColumnTypeKey/NullDescription
license_idVARCHAR(50)PK발급 고유 식별자
🔗 package_idVARCHAR(50)FK사용 권한을 부여받는 패키지
🔗 device_idVARCHAR(50)FK / NULL등록/할당된 디바이스 ID
serial_keyVARCHAR(100)UNIQUE사용자가 입력하는 16자리 등 시리얼키
auth_modeENUMNOT NULLALWAYS(상시), ONE_TIME(1회성)
statusENUMNOT NULLUNREGISTERED, REGISTERED, REVOKED
expires_atDATETIMENULL만료 일자 (구독형일 경우)
